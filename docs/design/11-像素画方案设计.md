# 像素画（Pixel Art）方案设计

> 文档目的：定义成就图标从像素数据存储到双端渲染的完整方案
> 最后更新：2026-05-19

---

## 一、格式定义

### 多分辨率 YAML 存储格式

每个成就图标存储 3 种分辨率（48×48 / 128×128 / 256×256），分别用于不同场景：

| 分辨率 | 用途 | 终端行数 | 数据量 |
|--------|------|---------|--------|
| **48×48** | 终端渲染、成就弹窗 | 24 行(半块) | 2304 字符 |
| **128×128** | Dashboard 卡片 | 64 行 | 16384 字符 |
| **256×256** | 详细信息页、放大视图 | 128 行 | 65536 字符 |

```yaml
- id: first_contact
  name: "Hello World"
  icon: "🚀"                              # 等宽字体的 fallback
  pixel_art:
    48:
      palette:
        - "⬛"                              # index 0: transparent
        - "#ff6b35"                         # index 1
        - "#ffd700"                         # index 2
        - "#8b4513"                         # index 3
      data:
        - "000000001111000000000000000000000000000000000000"
        - "00000001122110000000000000000000000000000000000"
        ...  # 48 rows × 48 chars
    128:
      palette:
        - "⬛"
        - "#ff6b35"
        ...
      data:
        - "000..."  # 128 rows × 128 chars
    256:
      palette:
        - "⬛"
        - "#ff6b35"
        ...
      data:
        - "000..."  # 256 rows × 256 chars
```

**设计决策**：
- `palette` 索引使用字符 `0-9` + `A-Z`，支持 ≤35 色（共 36 个索引，0=透明）
- 索引 `0` 固定为透明（渲染时跳过该像素）
- Palette 颜色按像素频率降序排列（最常见颜色 → 最小索引），最大化量化精度
- 各分辨率的 palette 独立（量化粒度不同，颜色分布有异）

### 坐标映射

```
48×48:
  data[0]:  "0000...00111..."   ← 像素行 0 (y=0), 48 字符
  data[1]:  "0000...01122..."   ← 像素行 1 (y=1)
  ...
  data[47]: "0000...00000..."   ← 像素行 47 (y=47)

palette[0] = ⬛ (transparent)
palette[1] = #ff6b35
palette[2] = #ffd700
...
palette[Z] = ...                 # 最多 35 色
```

## 二、输入源与生成工具

### 输入源：两种路径

| 来源 | 格式 | 适用场景 | 特点 |
|------|------|---------|------|
| **Phosphor 图标** | SVG (bold) | 抽象概念（工具、系统、动作） | 单色 → 自动着色 |
| **AI 生图** | PNG | 具体对象（物品、场景） | 彩色 → 直接量化 |

PNG 输入由 nano banana 等 AI 工具生成（用户提供详细描述），输出 512×512 左右的正方形彩色图。

### 生成工具：img-to-pixelart.mjs

```bash
# 用法
node img-to-pixelart.mjs <image-path> [--theme T] [--sizes 48,128,256]

# SVG 输入（单色 → 自动着色）
node img-to-pixelart.mjs icon.svg --theme fire --sizes 48,128,256

# PNG 输入（彩色 → 直接量化）
node img-to-pixelart.mjs ai-generated.png --sizes 48,128,256

# 主题：fire ocean gold neon nature royal cyber sunset emerald ruby steel amber
```

**处理管线**：

```
输入
  ├── SVG → sharp render → 单色检测 → 距离渐变着色 → 量化 → 多分辨率 YAML
  └── PNG → sharp render → 直接量化 → 多分辨率 YAML
```

**SVG 距离渐变着色**（monochrome → colored）：

1. 渲染为 48×48（或 128×128 / 256×256）灰度 mask
2. 2-pass Manhattan 距离变换：计算每个填充像素到最近背景的距离 → "内部深度"
3. 深度映射到 5 色渐变主题：edge(0) = shadow → center(max) = highlight
4. 边缘 1px 保留深色（shadow），内部渐亮，形成立体感

**PNG 直接量化**（color → palette + data）：

1. sharp 直接渲染到目标分辨率
2. 统计所有不透明像素的颜色频率
3. 取 Top 35 色作为 palette（按频率降序）
4. 每个像素映射到最近 palette 色索引

## 三、渲染

### 终端渲染

使用 **Unicode 半块字符** + ANSI 24-bit 真彩色：

```
每 2 行像素 = 1 行终端字符
48 像素行 → 24 行终端输出（48 列宽）
```

```
像素行 y:    ████▀▀▀▀    ← u_color = palette[upper[x]]
像素行 y+1:  ▀▀▀▀▄▄▄▄    ← l_color = palette[lower[x]]

u有且l有 → █ (全块 fg=u bg=l)
u有 l无   → ▀ (上半块 fg=u)
u无 l有   → ▄ (下半块 fg=l)
都无      →   (空格)
```

**Claude Code 注意**：终端预览输出较宽，CC 默认折叠 ANSI 块级输出，需按 `ctrl+o` 展开查看。

### Dashboard 渲染

同一份 YAML 数据，前端转 SVG（`pixel_art.{resolution}.data` 中每字符 = 1 像素）：

```typescript
function renderPixelArtSVG(pixelArt: PixelArt, size: 48|128|256): string {
  const cellSize = 2; // 每个像素 = 2px SVG
  const w = size;
  const data = pixelArt[size].data;
  const palette = pixelArt[size].palette;

  let rects = "";
  for (let y = 0; y < w; y++) {
    for (let x = 0; x < w; x++) {
      const c = data[y][x];
      if (c === "0") continue;
      const ci = parseInt(c, 36); // 0-9A-Z → 0-35
      rects += `<rect x="${x*cellSize}" y="${y*cellSize}" width="${cellSize}" height="${cellSize}" fill="${palette[ci]}"/>`;
    }
  }
  return `<svg viewBox="0 0 ${w*cellSize} ${w*cellSize}">${rects}</svg>`;
}
```

SVG 可缩放到任意尺寸，天然抗锯齿。

## 四、风格规范

### 通用风格

```
分辨率:  48×48 / 128×128 / 256×256
调色板:  ≤ 35 色（含透明）
      36 个索引（0-9 + A-Z），0 = transparent

常见技巧：
  - 45° 对角线用 2:1 阶梯模拟
  - 小物体直接省略，保持构图简洁
  - 用透明像素做边缘抗锯齿
```

### 各分类构图建议

| 分类 | 构图思路 | 参考色系 |
|------|---------|---------|
| Onboarding | 小物品（卷轴、信封、星星、光标） | 暖色调 |
| Milestone | 数字 + 容器（奖杯、盾牌、计量条） | 金/银/铜 |
| Skill | 工具（锤子、齿轮、闪电、放大镜） | 蓝/紫色系 |
| Style | 抽象（月相、沙漏、节奏条、笔触） | 渐变 |
| Tool Mastery | 对应工具的物化（终端、书、代码括号） | 绿/青色 |
| Workflow | 流程（箭头、循环、分支、管道） | 蓝/绿色 |
| Creator | 制作（锤+星、魔棒、齿轮） | 紫色/金色 |
| Hidden | 谜题（问号、钥匙、锁、面具） | 暗色 + 高对比 |
| Challenges | 奖牌、桂冠、秒表、火焰 | 红/金色 |
| Community | 多人（握手、地球、分享箭头） | 蓝/绿色 |

## 六、占位与降级

成就 YAML 中 `pixel_art` 为**可选字段**。不存在时降级使用 `icon`（emoji）：

```yaml
# 有像素画
- id: first_contact
  icon: "🚀"
  pixel_art:
    48: { palette: [...], data: [...] }
    128: { palette: [...], data: [...] }
    256: { palette: [...], data: [...] }

# 无像素画（降级到 emoji）
- id: tool_time
  icon: "🛠️"
  # 无 pixel_art → 终端/Dashboard 展示 emoji
```
