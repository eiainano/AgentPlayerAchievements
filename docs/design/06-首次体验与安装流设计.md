># 首次体验与安装流设计（First Run Flow）

> 文档目的：定义用户从安装 agpa 到首次看到成就弹窗（"Aha Moment"）再到打开 Dashboard 的完整体验流程

---

## 一、用户旅程总览

```
brew install agpa   agpa init --tool cc   启动 Agent             首次 poll()              agpa dashboard
      │                    │                   │                       │                       │
      ▼                    ▼                   ▼                       ▼                       ▼
┌───────────┐     ┌───────────────┐     ┌───────────┐          ┌──────────────┐          ┌──────────┐
│ 安装二进制  │ ──→ │ 注册 MCP + 注入│ ──→ │ MCP 自动  │ ──→ 成就评估触发 │ ──→      │ 浏览器打  │
│ + 创建目录  │     │ 事件采集配置   │     │ 拉起      │     │ 条件满足→弹出   │          │ 开查看    │
│ + 首次提示  │     │ + 首次成就清单 │     │ 首个事件   │     │ 终端弹窗展示   │          │ 完整面板  │
└───────────┘     └───────────────┘     └───────────┘          └──────────────┘          └──────────┘
      │                   │                   │                       │                       │
     ~2s                ~1s                agent 启动时             第一轮对话后            用户主动

```

**关键指标**：从安装到首次看到成就弹窗 < agent 的首轮响应时间（重叠进行，零额外延迟感）。

## 二、Step 1: `brew install agpa`

### 安装内容

```bash
brew install agpa
```

安装后系统获得：

```
/usr/local/bin/agpa          # 主二进制（~5MB 单文件）
/usr/local/share/agpa/       # 资源文件
  ├── achievements.yaml      # 默认成就定义（109 个）
  ├── default-profile.json   # 默认 profile 模板
  └── display-instructions/  # 各工具指令模板
      ├── claude-code.md
      ├── kilo-code.md
      ├── hermes.md
      ├── opencode.md
      └── openclaw.md
```

### 安装后首次行为

**不自动创建 `~/.agent-achievements/`**。只在用户第一次运行 `agpa init` 时创建。原因：

1. **不要污染用户文件系统**——用户可能只是 `brew list` 看一下
2. **`~/.agent-achievements/` 的创建 = 用户明确决定使用 AGPA**
3. `brew install` 保持纯净：写 `/usr/local/` 范围，不碰 `~`

### 安装后立即展示的内容

```
🏆 Agent Player Achievements installed!

Get started:
  agpa init --tool claude-code    # Link with Claude Code
  agpa init --tool kilo-code      # Link with Kilo Code
  agpa init --tool opencode       # Link with OpenCode
  agpa init --tool openclaw       # Link with OpenClaw
  agpa init --tool hermes         # Link with Hermes Agent

Need help?
  agpa --help                     # Full command reference
  agpa dashboard                  # Open achievement dashboard
```

### 升级场景

```bash
brew upgrade agpa
```

- 只覆盖二进制 + `/usr/local/share/agpa/` 下的资源文件
- **不覆盖** `~/.agent-achievements/achievements.yaml`（用户自定义的成就定义）
- 如果新版有更新的 `achievements.yaml`，拷贝为 `achievements.yaml.new`，让用户手动合并
- `~/.agent-achievements/state.json` 自动迁移（`version` 字段控制）

## 三、Step 2: `agpa init --tool <xxx>`

### 执行流程

```
agpa init --tool claude-code

  Step 2.1  检查 ~/.agent-achievements/ 是否存在
             不存在 → 创建目录 + 初始化默认文件

  Step 2.2  读取 achievements.yaml
             解析 109 个成就定义 + 9 个套装定义
             校验 YAML 格式 → 写初始 state.json

  Step 2.3  检测指定工具的配置文件路径
             CC:   ~/.claude/settings.json
             Kilo: VS Code settings 中的 kilocode.mcpServers
             Hermes: cli-config.yaml
             OC:   opencode.json / TOML
             OpenClaw: ~/.openclaw/openclaw.json

  Step 2.4  检测目标文件是否存在
             ✅ 存在  → 读取现有配置，追加 MCP 和 Hook
             ❌ 不存在 → 输出错误提示（见下文错误处理）

  Step 2.5  注入 MCP Server 配置
             CC:   mcpServers["agent-achievements"] = { command: "agpa", args: ["mcp"] }
             Kilo: 等同 CC，写入 VS Code settings
             ...

  Step 2.6  注入 Hook / 事件采集配置
             CC:   8 个 Hook（SessionStart / SessionEnd / TaskCreated / TaskCompleted / PreToolUse / PostToolUse / PostToolUseFailure / PostCompact）
             其他: 写入 instructions 文件（AGENTS.md / .opencode/rules 等）

  Step 2.7  注入 achievement poll 指令
             CC:   CLAUDE.md 追加 <achievement_display> 指令块
             Kilo: AGENTS.md 追加
             ...

  Step 2.8  输出成功提示 + 首次成就预告
```

### 错误处理

#### 情况 1：目标工具未安装

```bash
$ agpa init --tool kilo-code
  ✖ Kilo Code not found.
    Install it first: https://marketplace.visualstudio.com/items?itemName=...
```

检测方式：`init` 时检查该工具的配置文件路径是否存在。

#### 情况 2：配置文件格式异常

```bash
$ agpa init --tool claude-code
  ⚠ ~/.claude/settings.json exists but is not valid JSON.
    Backup created at ~/.claude/settings.json.agpa.bak
    Continue with a fresh config? [Y/n]
```

检测方式：解析 JSON，parse error 时提示用户。用户确认后覆盖。

#### 情况 3：已有 MCP Server 注册

```bash
$ agpa init --tool claude-code
  ℹ agent-achievements MCP server already registered.
    Updating configuration...
    ✅ Done (no changes needed)
```

幂等保证：检测 `mcpServers` 中已有 `agent-achievements`，对比配置值，一致则跳过，不一致则更新并备份旧配置。

#### 情况 4：文件权限不足

```bash
$ agpa init --tool claude-code
  ✖ Cannot write to ~/.claude/settings.json
    Permission denied. Check file ownership.
    Hint: ls -la ~/.claude/settings.json
```

检测方式：打开文件前 `access(F_OK | W_OK)`。失败时给出明确指引，不静默降级。

### 幂等性保证

```
首次 init:   创建目录 + 写入配置 + 注入指令（全量操作）
再次 init:   检测已有 → 对比目标值
  MCP 配置一致  → 跳过，无操作
  MCP 配置不同  → 更新，备份旧配置到 .agpa.bak
  指令文件一致  → 跳过
  指令文件不同  → 追加（不删除原有内容，只添加缺失部分）
  成就数据存在  → 不重置 state.json（不做破坏性操作）
```

**核心原则**：`agpa init` 永远不会重置已有成就数据。用户需要重置时主动使用：

```bash
agpa reset           # 重置所有成就数据（确认提示）
agpa reset --hard    # 重置 + 清除 event.log（再次确认）
```

### multi-tool 注册

用户可以为多个工具注册同一套成就系统：

```bash
agpa init --tool claude-code
agpa init --tool kilo-code    # 共享同一份 ~/.agent-achievements/
agpa init --tool opencode     # 跨工具累积成就
```

每次 init 各自注入对应工具的配置和指令。event.log 和 state.json 共享（同 profile 下）。

## 四、Step 3: 首次 Agent 启动

### 启动时序

```
用户输入 prompt
      │
      ▼
Agent 启动 → 读取 CLAUDE.md / AGENTS.md
      │
      ├── MCP Server 自动 spawn
      │   agpa mcp
      │      │
      │      ├── 创建状态目录（如果还没有）
      │      ├── 读 event.log 水线 → 0（首次）
      │      ├── 检查 rarity_cache（首次为空，跳过）
      │      └── 等待 tool 调用
      │
      ├── 用户继续输入 prompt
      │
      ├── Agent 开始执行
      │   ├── 会话建立 → SessionStart Hook 触发
      │   │       → agpa track session.start → event.log
      │   │
      │   ├── 用户输入 prompt → TaskCreated Hook 触发
      │   │       → agpa track task.start → event.log
      │   │
      │   ├── 第一次 tool call
      │   │       ↓
      │   │   PreToolUse Hook 触发
      │   │       → agpa track tool.call → event.log
      │   │       ↓
      │   │   tool 执行完成
      │   │       ↓
      │   │   PostToolUse Hook 触发
      │   │       → agpa track tool.complete → event.log 追加第一行
      │   │
      │   ├── 继续对话...
      │   │
      │   └── 第一轮结束前
      │           ↓
      │       CLAUDE.md 指令触发：
      │       "Before responding, call achievement.poll()"
      │           ↓
      │       MCP tool: achievement.poll()
      │           ↓
      │       Evaluator 首次扫描 event.log
      │           ↓
      │       成就条件评估
      │
      ▼
  Agent 输出响应（含成就弹窗）
```

### 首次 poll() 内部时序

```
achievement.poll()  # 由 agent 在每轮结束时调用
      │
      ├── PendingQueue 检查
      │   └── 空 → 触发 evaluate()
      │
      ├── evaluate()
      │   ├── flock(LOCK_EX) state.json
      │   ├── 读 watermark = 0
      │   ├── 扫 event.log 从第 0 行到 EOF
      │   │     ↓
      │   │   找 first_contact 条件：event_type == "conversation.message" >= 1
      │   │   首次对话 → 找到 → 条件满足
      │   │     ↓
      │   │   找 three_s_company: conversation.message >= 10 → 否，跳过
      │   │   找 tool_time: tool.complete >= 1
      │   │     如果已有 tool call → 条件满足
      │   │   其他成就 → 逐一检查
      │   │
      │   ├── 解锁成就 → 写入 state.json → 加入 pending
      │   ├── 更新 watermark = N
      │   ├── 写 state.json
      │   └── flock(LOCK_UN)
      │
      ├── 读 pending 队列
      │   └── 有成就 → 返回 [{id: "first_contact", ...}]
      │
      └── 返回给 agent
```

### 首次可能解锁的成就

第一次 poll() 时，根据用户行为，可能同时解锁多个 Onboarding 成就：

| 成就 | 触发条件 | 首次 poll() 时状态 |
|------|---------|-------------------|
| `first_contact` | 首条 conversation.message | ✅ 必然解锁 |
| `tool_time` | 首次 tool.complete | ✅ 如果用户发了 prompt |
| `permission_granted` | 首次 adjust permission mode | ⚠️ 用户操作过才解锁 |
| `read_the_manual` | 首次打开帮助 | ⚠️ 如果用户打过 /help |
| `hat_trick` | 3 次 conversation.message | ⚠️ 如果对话超过 3 轮 |
| `command_apprentice` | 首次 slash command | ⚠️ 如果用户用了命令 |
| `model_hopper` | model.switch | ⚠️ 如果用户切过模型 |

设计考虑：第一次 poll() **可以同时返回多个成就**，PendingQueue 的 `urgent` 机制在此处不启用（首次展示大于 > 3 个也是合理的）。

## 五、Step 4: 终端弹窗（Aha Moment）

### 第一个成就弹窗

用户的第一条 prompt 发出后，Agent 返回响应时，输出中会包含这个弹窗：

```
╭──────────────────────────────────────────────╮
│  🏆 Achievement Unlocked!                    │
│                                              │
│  Hello World                                 │
│  在 agent 里完成了第一次对话。你已             │
│  经走出了探索 AI 的第一步。                   │
│                                              │
│  Rarity: Common · Starting Line              │
│  XP +50                                      │
│                                              │
│  Set: The Beginning (1/11) █░░░░░░░░░░       │
╰──────────────────────────────────────────────╯

[Agent 的正式输出开始...]
```

如果同时解锁多个成就，按以下顺序排列：

```
╭──────────────────────────────────────────────╮
│  🏆 2 Achievements Unlocked!                 │
│                                              │
│  ✦ Hello World         +50 XP                │
│    Common · Set: The Beginning 1/11          │
│                                              │
│  ✦ Tool Time            +50 XP               │
│    Common · Set: The Beginning 2/11          │
│                                              │
│  🎯 Next: "Three's Company" (8 more to go)   │
│      Chat 10 times to unlock                 │
╰──────────────────────────────────────────────╯
```

### 弹窗定位

弹窗出现在 Agent 响应正文**之前**（上方），用户阅读回复前先看到成就。

```
用户 prompt: "用 Python 写一个斐波那契数列"

Agent 响应:
  ┌─────────────────────────────────────┐
  │ 🏆 Hello World Unlocked!            │  ← 成就弹窗（最先看到）
  │ 🏆 Tool Time Unlocked!              │
  └─────────────────────────────────────┘

  > 斐波那契数列有多种实现方式。以下是...
                                        ← 正式回复
```

### Aha Moment 设计要点

1. **第一次弹窗必须出现在第一个回复中**，不让用户等待
2. **弹窗内容简洁**：成就名 + XP + 进度，不展示完整 YAML 定义
3. **弹窗后有清晰的"下一步"线索**：`Next: "Three's Company" (8 more to go)`
4. **不打断流**：弹窗在输出中占 < 10 行，用户扫一眼就过了
5. **XP 数字**即使在后端很小，也让用户产生"我在积累"的感觉

### 副通道：OS 通知

首次解锁同时弹出系统通知：

```
macOS Notification Center:
  ┌─────────────────────────────────┐
  │ 🏆 Achievement Unlocked!       │
  │ Hello World                    │
  │ Common · The Beginning 1/11    │
  │                                │
  │ 在 agent 里完成了第一次对话。   │
  └─────────────────────────────────┘
```

**首次弹窗**：如果用户允许，解锁时可以触发系统通知的声音效果（macOS 使用 `sound name "Glass"`），之后不再重复声音。

### 空状态 & 引导设计

**成就网格中的已解锁**：
- 已解锁成就正常显示（icon + 名称 + 稀有度标签）
- 未解锁成就显示 🔒 图标，半透明，hover 时显示类别提示

- Dashboard 顶部 banner 提示当前进度（只在前 3 次打开时展示，或成就要到 10+ 后自动消失）

- 不展示全球统计（数据不够，至少 5 个成就解锁后启用）
- 不展示活跃的"推荐目标"（数据不够，10 个成就解锁后启用）

## 七、边缘情况

### 情况 A：用户先启动 Agent，后执行 `agpa init`

```
时序：
  Agent 启动（没有 agpa MCP） → MCP 不存在，agent 正常运行
  → 用户执行 agpa init
  → 下次 Agent 启动 → MCP 自动拉起
  → event.log 从空开始

影响：init 之前的 Agent 使用记录不追溯。
       成就从 init 之后的第一次使用开始计数。
```

**设计决策**：
- 不追溯历史事件。`agpa init` 那一刻就是计时起点
- 用户如果希望追溯，需要手动导入（`agpa track` 单独追加历史事件）
- 初始两个成就 `first_contact` 和 `tool_time` 设计门槛极低（≥1 次事件），init 后的第一次使用必然解锁

### 情况 B：用户从未执行 `agpa init`，直接 `agpa mcp`

```
agpa mcp 启动时检测：
  ~/.agent-achievements/state.json 不存在
    → 自动创建（等同 init 的效果，但只做最小初始化）
    → 不注入工具配置、不写 instruction 文件
    → MCP Server 正常提供 tool 调用

影响：
  工具可以调用 achievement.track() 和 achievement.poll()
  但因为没有 instruction 文件：
    → agent 不会主动调用 poll()
    → 成就仍然会解锁并加入 pending
    → 但不会在终端弹出
    → 唯一查看方式：agpa dashboard

解决方案：
  如果用户后来执行 agpa init，instruction 文件补上
  已经解锁的成就不会丢失
```

**设计决策**：`agpa mcp` 是唯一一个不需要 `init` 就能运行的模式。用户可以通过纯 MCP 方式使用系统（虽然体验打折），但不是硬性依赖。

### 情况 C：卸载 agpa

```bash
brew uninstall agpa
```

**卸载后影响**：

| 资源 | 卸载后状态 | 备注 |
|------|-----------|------|
| `/usr/local/bin/agpa` | 删除 | — |
| `~/.agent-achievements/` | **保留** | 用户数据，不自动删除 |
| MCP Server 配置 | 残留 | 如果 agent 启动时找不到 agpa 命令会有错误日志 |
| instruction 文件 | 残留 | 不会影响 agent（poll 调用会因 MCP 工具不存在的空返回优雅降级） |

**手动清理**：

```bash
# 清理 agpa 所有痕迹
agpa uninstall          # 删除 ~/.agent-achievements/
agpa uninstall --all    # + 清理所有工具的 MCP 配置
                        # + 清理 instruction 文件中的 poll 指令
```

`brew uninstall` 之后这些命令仍然可用（设计上 `uninstall` 子命令是内嵌在主二进制中的独立逻辑）。

### 情况 D：`agpa init` 重复执行

```
第一次：  agpa init --tool claude-code  →  写入 MCP + Hook + CLAUDE.md
第二次：  agpa init --tool claude-code  →  检测已存在，无操作（幂等）
第三次：  agpa init --tool claude-code  →  同

同 profile 不同 tool：
第一次：  agpa init --tool claude-code  →  CC 配置
第二次：  agpa init --tool kilo-code    →  Kilo 配置（共享成就数据）
第三次：  agpa init --tool opencode     →  OC 配置

不同 profile：
第一次：  agpa init --tool cc --profile work    →  ~/.agent-achievements/work/
第二次：  agpa init --tool cc --profile personal →  ~/.agent-achievements/personal/
```

### 情况 E：event.log 首次为空/水线溢出

```
首次 poll()：
  watermark = 0
  event.log 为空 → 没有新事件 → 跳过评估
  → 无成就解锁

正常情况：首次 poll() 时用户已经有对话 → event.log 有内容
极低概率：用户启动 agent 后立即 / 在第一个回复前触发了 poll()
```

**设计决策**：空 event.log 的 poll() 返回空 pending，不做任何副作用。agent 的下一次 poll() 自然评估。

### 情况 F：安装后立刻 `agpa dashboard`（无成就数据）

```
Dashboard 启动
  → 读 state.json
  → 全部成就 locked
  → 展示空状态

展示内容：
  - Level 1，XP 0/100
  - 展示柜全部 🔒
  - 成就网格全部 🔒
  - Timeline 空
  - 顶部 banner: "💡 第一次使用？你需要先设置 agpa init，然后启动你的 Agent 开始工作"
```

**设计决策**：unlocked=0 的 Dashboard 不是无意义展示。用户可以通过浏览成就列表来"种草"——看到感兴趣的成就，从而有动力去探索对应功能。

## 八、First Run 成就追踪

### 首批解锁成就设计

首次使用的预期解锁曲线：

```
第一次对话完成     → first_contact ✅     → XP +50
第一次 tool call   → tool_time ✅         → XP +50
第 3 次对话        → hat_trick ✅         → XP +50（如果同时达成）
第 10 次对话       → three_s_company ✅   → XP +50
首次 permission    → permission_granted ✅→ XP +50（如果用户操作过）
首次 slash command → command_apprentice ✅→ XP +50（如果用户操作过）
```

**预期首次 session 可能解锁 2-5 个成就**（黄金路径）。

### 首次 XP 曲线

```
Level 1 → Level 2: 100 XP
  首次 session 解锁 2-5 个成就 → 100-250 XP
  → 大概率在首次使用后直接升到 Level 2

Level 2 → Level 3: 200 XP
  第 2-3 次 session → 解锁更多 Onboarding 成就 + 挂到中级成就
  → XP 稳步积累
```

XP 曲线的陡峭度设计：前 10 级平滑快速（每 1-2 次 session 升一级），10 级后逐渐平缓。让新用户在最初几次使用中持续获得升级反馈。
