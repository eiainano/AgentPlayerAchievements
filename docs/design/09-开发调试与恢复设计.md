### 1.2 Dev 工作流

```bash
# 开发
npm run dev           # ts watch → node dist/index.js
npm run dev -- track  # 调试某个子命令

# 测试
npm test              # 跑所有单元测试
npm run test:watch    # watch 模式
npm run test:integ    # 集成测试（需要临时文件系统）

# 构建
npm run build         # tsc → dist/
npm pack              # 本地 .tgz 包
```

**测试策略**：

```
单元测试（~70%）:
  - Evaluator: 注入 fixture event.log，验证成就解锁状态
  - EventBuffer: 验证 O_APPEND 写入、fsync 策略切换、水线更新
  - validate: 合法/非法 event_type 的通过/拒绝
  - lock: flock 竞争场景的正确释放

集成测试（~20%）:
  - track → event.log → evaluate → poll 完整链路
  - 双进程并发写入 + 同时 poll
  - state.json 损坏后修复

端到端（~10%）:
  - npm run dev 起的进程，用 MCP client 连上发 tool 调用
  - 验证 terminal ANSI 输出格式（mock terminal）
```

**确定性测试技巧**（Evaluator 测试的关键难点）：

```typescript
// 不依赖真实文件，用 mock EventBuffer + 预置事件列表
const events = [
  { event_type: "conversation.message", timestamp: "2026-05-01T10:00:00Z", ... },
  { event_type: "tool.complete", timestamp: "2026-05-01T10:00:05Z", ... },
];
const evaluator = new CounterEvaluator({ type: "counter", event: "conversation.message", operator: ">=", value: 1 });
const result = evaluator.evaluate(events);
assert(result.unlocked === true);
```

## 二、调试

### 2.1 日志体系

```bash
# 日志文件位置
~/.agent-achievements/
├── event.log             # 事件流（JSONL，主要的"审计日志"）
└── debug.log             # 运行日志（循环写入，保留最近 1MB）
```

**日志级别**：

| 级别 | 用途 | 示例 |
|------|------|------|
| `debug` | 追踪每步操作（默认关闭，`--verbose` 开启） | `[debug] track: building event for tool.complete` |
| `info` | 正常操作记录 | `[info] track: appended event a1b2c3 to event.log` |
| `warn` | 非致命异常 | `[warn] track: tool_source unknown, using "unknown"` |
| `error` | 操作失败 | `[error] track: write failed: ENOSPC` |

**激活方式**：

```bash
agpa --verbose mcp           # MCP 模式输出 debug 日志到 stderr
agpa --verbose track ...     # CLI 模式
agpa doctor --logs           # 查看最近 debug.log
```

debug.log 循环写入策略：
- 单文件最大 1MB
- 超出后截断前半部分（保留最近 ~500KB）
- 不轮转（避免用户目录被日志填满）

### 2.2 agpa doctor 诊断命令

```bash
agpa doctor
```

诊断项：

```
┌─────────────────────────────────────────────┐
│  AGPA Doctor                                │
│                                             │
│  检查项：                                    │
│                                             │
│  ✅ ~/.agent-achievements/ 目录存在          │
│  ✅ event.log 可写（5572 行，最近修改 2min ） │
│  ✅ state.json 可读（版本 2，52 成就已评估）  │
│  ✅ state.json 锁状态：未锁定                │
│  ⚠  achievements.yaml 使用默认定义（未自定义）│
│  ❌ MCP 配置未找到（预期：~/.claude/settings.json）│
│                                             │
│  建议：                                      │
│    - 执行 agpa init --tool claude-code       │
│      注册 MCP server                         │
│                                             │
│  Debug logs:                                 │
│    ~/.agent-achievements/debug.log           │
│    (上次运行 agpa --verbose doctor)           │
└─────────────────────────────────────────────┘
```

**诊断项列表**：

```
1. 目录完整性      → ~/.agent-achievements/ 存在且含必要文件
2. event.log       → 存在、可写、行数 > 0、最近修改时间
3. state.json      → 存在、可读、版本兼容、有已解锁成就
4. 锁状态          → state.json 当前未被 flock 锁定
5. achievements.yaml → 存在、YAML 可解析
6. MCP 配置        → 各工具的 MCP Server 配置是否存在
7. instruction 文件 → CLAUDE.md / AGENTS.md 含 poll 指令
8. 全局统计 opt-in → 是否已开启
```

**每一项都有单独的 exit code**，方便脚本调用时判断：

```bash
agpa doctor --check event-log   # 只查 event.log，exit 0/非0
agpa doctor --check mcp-config  # 只查 MCP 配置
```

### 2.3 常见问题排查

```
问题 1：成就没有弹出
  → agpa doctor 检查 MCP Server 是否注册
  → 检查 ~/.claude/settings.json 中 mcpServers 段
  → 检查 CLAUDE.md 是否包含 <achievement_display> 指令
  → 重启 Agent

问题 2：成就不推进
  → 检查 event.log 是否有新行（tail -f event.log）
  → 如果有新事件但成就不动 → poll 可能没被调用
  → 检查指令文件中是否有 poll() 调用

问题 3：EventBuffer 写入报错
  → 检查 ~/.agent-achievements/ 磁盘空间
  → 检查 event.log 权限（ls -la event.log）
  → agpa doctor --check event-log

问题 4：Dashboard 打不开
  → 端口 3867 被占用？→ lsof -i :3867
  → agpa dashboard --port 3868 换端口
```

## 三、恢复

### 3.1 state.json 损坏

```json
// 损坏场景
// 写 state.json 时进程崩溃 → 文件只有半截
{
  "version": 2,
  "achievements": {
    "first_contact": {          ← 截断在这里
```

**恢复策略 A：从 event.log 重建（推荐）**

```bash
# 自动恢复
agpa doctor --fix

# 手动恢复步骤：
# 1. 备份损坏文件
cp state.json state.json.corrupt

# 2. 从 event.log 全量扫描重建 state.json
agpa track --replay event.log
# → evaluator 重跑所有事件 → 从零重建 state.json
# → 保留原有成就定义，仅丢失上次 poll() 到 crash 之间的增量评估

# 3. 验证
agpa doctor --check state-json
```

**恢复策略 B：手动修复（用户自己修复 JSON）**

```bash
# 如果只是最后一个 } 丢失：
echo '}' >> state.json
# 然后用 jsonlint 验证
```

### 3.2 event.log 截断

```
场景：磁盘满、文件系统错误 → event.log 最后几行损坏

Evaluator 扫描时遇到损坏行 → 跳过 + 记录行号

恢复：
  tail -n +<损坏开始行号> event.log > event.log.fixed
  mv event.log.fixed event.log
  # 或直接用 agpa doctor --fix 自动修复
```

### 3.3 锁死

```
场景：进程崩溃时持有 state.json 的 flock

flock 的特性：
  → 持有锁的进程退出时，OS 自动释放
  → 所以"锁死"在正常 OS 行为下不会发生

极低概率（NFS / 分布式文件系统等情况）:
  agpa doctor --fix          # 检查锁状态，提示用户手动处理
  lsof ~/.agent-achievements/state.json  # 查看哪个进程持有
  kill -9 <PID>              # 强制释放
```

### 3.4 数据导出/迁移/重置

```bash
# 导出所有成就数据
agpa export > agpa-backup.json
# 包含：state.json + history.json + profile.json

# 迁移到新机器
agpa import agpa-backup.json

# 重置成就（不清除 event.log，可重建）
agpa reset
# 确认提示：Are you sure? This will reset all achievement progress. (y/N)

# 硬重置（清空所有数据，从头开始）
agpa reset --hard
# 二次确认：This will DELETE all event history. Continue? (y/N)
```

**`agpa export` 导出格式**：

```json
{
  "exported_at": "2026-05-18T...",
  "agpa_version": "0.1.0",
  "data": {
    "state": { ... },
    "history": [ ... ],
    "profile": { ... },
    "stats": { ... }
  }
}
```

**迁移原则**：
- event.log **不导出**（太大了，而且重建后不需要历史事件）
- 只导出评估后的状态 + 解锁历史
- 导入后，成就进度保持导出时的状态，不再重新评估

### 3.5 恢复设计总览

```
                    ┌────────────────┐
                    │  数据损坏检测   │
                    │ (agpa doctor)  │
                    └───────┬────────┘
                            │
              ┌─────────────┼──────────────┐
              ▼              ▼              ▼
    ┌────────────────┐ ┌──────────────┐ ┌──────────────┐
    │ state.json     │ │ event.log    │ │ 锁死         │
    │ 损坏           │ │ 截断         │ │              │
    └───────┬────────┘ └──────┬───────┘ └──────┬───────┘
            │                 │                │
            ▼                 ▼                ▼
    ┌────────────────┐ ┌──────────────┐ ┌──────────────┐
    │ 从 event.log   │ │ tail -n+x    │ │ OS 自动      │
    │ 全量重建        │ │ 截断后半部分  │ │ 释放锁       │
    │ (agpa replay)  │ │ (或自动修复)  │ │ (进程退出时)  │
    └────────────────┘ └──────────────┘ └──────────────┘
            │
            ▼
    ┌────────────────┐
    │ 用户数据不丢失   │
    │ 仅丢失最后一次  │
    │ poll() 到 crash │
    │ 之间的增量评估  │
    └────────────────┘
```
