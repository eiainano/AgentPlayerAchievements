## Top 10 TODO

### 1. 推荐系统 2.0——从"近锁"升级为"值得玩什么"

**状态**: 未开始 | 优先级: ★★★★★ | 预估: 中（~2-3 天）

这是昨天 #1，**什么都没变，依旧 #1**。Questline 给了"结构"，推荐系统给"指引"。两者互补。

**核心设计：**
- **Near Win**：离解锁最近的 3 个（已有 progress nudge 雏形，需改造）
- **Discovery**：推荐一个你还没体验过的功能/玩法
- **Challenge**：推荐一个略高难但有意思的目标
- **Surprise**：隐藏成就 hint（不剧透）

**为什么 Questline 落地不降低此优先级**：Questline 是**静态路径**，推荐是**动态感知**。Questline 不需要知道用户此刻的情况；推荐引擎需要。这是完全不同的产品能力。

**价值**：这是最直接提升"每天打开 AGPA 看看有什么可玩"的功能。

---

### 2. 成就可解释层 (Explain Layer)

**状态**: 未开始 | 优先级: ★★★★★ | 预估: 中（~2-3 天）

**此条昨天是 #6，今天上升到 #2。原因：**

刚刚在 comprehensive code review 中修复了 14 个 HIGH Bug（包括 evaluator 绕过、窗口无视、filter 静默放行等）。这些 Bug 的存在说明：

1. 即使评估引擎看似正常工作，边界情况下可能有错误
2. 用户面对 218 个成就，没有办法区分"我没达标"和"系统没算对"
3. 如果用户不信任进度，整个成就系统的产品价值会打折扣

**建议能力（Dashboard Modal／CLI explain 子命令）：**
- 已命中条件 ✓
- 当前计数 / 进度
- 还差多少 / 下一次触发需要什么
- 当前窗口范围
- 哪些事件被 filter 排除了
- Hidden 成就只给 hint 级解释

**价值**：这不是调试工具，是**用户信任层**。与推荐系统配合一起构成"知道差距、知道下一步"的产品闭环。

### 4. 文档真相源自动化

**状态**: 部分完成（手动同步） | 优先级: ★★★★☆ | 预估: 小（~1 天）

**此条昨天是 #10，今天上升到 #4。原因：**

昨天刚手工同步了所有文档中的数字（成就数、测试数、Condition Type 数等）。手工同步不可持续——今天 Banner 测试挂了后，issues-todo 中顶部的"1067 tests"已经不正确（实际 1064）。

**建议方案：**
- 新增一个 `npm run docs:verify` 命令
- 自动从代码源（YAML 成就数、vitest 输出、types.ts 的 Condition type 数）提取关键数字
- 与 docs/ 中的数字比对，不一致时报 warning / error
- 可选的 `--fix` 模式自动更新文档中的数字

**价值**：这会彻底消除"文档与代码不同步"的长期维护负担，而且能提前发现类似"测试挂了但文档没更新"的情况。

### 7. 下一波精品内容扩展（20-30 个）

**状态**: 未开始 | 优先级: ★★★☆☆ | 预估: 中（~2-3 天）

218 个成就已足够大，不追求机械扩容。但昨天完成的 **5 个 Questline**（Bug Hunter、Toolsmith、Builder、Night Shift、Polyglot）暴露了一个问题：

有些 stage 只有 2-3 个成就，路径感不够充实。

**建议优先补的内容类型：**
1. **Questline 填充**——为每个 questline 每个 stage 补到至少 4-5 个成就
2. **Emergent / sandbox 行为**——"原来这也算"型成就
3. **Challenge ladder**——阶梯式挑战，而非纯数字放大
4. **Playstyle branching**——不同使用风格走出不同路径

**价值**：精品内容比总量更能定义 AGPA 的品牌风格，同时也能让 Questline 更有"旅程感"。
