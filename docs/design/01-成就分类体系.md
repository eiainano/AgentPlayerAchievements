# 成就分类体系

## 成就定义语言

每个成就 = 一个叙事小品：

```json
{
  "id": "the_debugger",
  "name": "The Debugger",
  "description": "agent 代码报错，你让它自己修，修好了。这就是信任。",
  "icon": "🐛",
  "category": "skill",
  "rarity": "uncommon",
  "hidden": false,
  "set": "the_bug_catcher",
  "conditions": [
    { "type": "in_single_task", "events": ["test.fail", "agent.self_fix", "test.pass"], "order": true }
  ],
  "progress_trackable": true
}
```

原则：
- **名称**：有记忆点，梗/双关/幽默优先
- **描述**：告诉你「做了什么」，而非「需要做什么」
- **稀有度**：动态计算，基于全体用户持有率
- **隐藏**：未解锁时只显示「???」，解锁后揭晓

## 稀有度系统

不手动赋稀有度，动态计算：

```
Common:    持有率 > 50%
Uncommon:  持有率 20-50%
Rare:      持有率 5-20%
Epic:      持有率 1-5%
Legendary: 持有率 0.1-1%
Mythic:    持有率 < 0.1%

展示格式：
  "Mind Meld 🧠 — Uncommon (34.7% of users have this)"

稀有度历史：
  记录用户解锁时的稀有度
  → 「你在它还只有 2% 的时候就拿到了」——早期用户的成就更值钱
```

稀有度计算需要匿名统计上报（opt-in）。离线时使用 rarity_cache.json 的缓存值。
