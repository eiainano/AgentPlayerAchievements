# AGPA 接下来最有价值的 Top 10 TODO

> 日期：2026-06-10  
> 目的：基于当前项目状态、既有设计文档与已完成调研，提炼 AGPA 下一阶段**最值得投入**的 10 个方向。  
> 结论导向：不是“最容易做的 10 个”，而是“对产品价值、用户体感和长期演进最有帮助的 10 个”。

---

## 一、核心判断

AGPA 当前已经基本跨过“底层可用性”阶段：
- 5 个 Agent 工具双通道覆盖
- 208 个成就
- 1067 测试
- Dashboard 已有分享卡、动画、称号、徽章、响应式等完整体验雏形

因此，下一阶段最有价值的工作，**不再是继续证明系统能追踪、能评估、能展示**，而是：

> **把 AGPA 从“能记录成就的系统”，推进成“更像一个有成长路径、收藏欲、回访理由的游戏化产品”。**

换句话说，接下来的重点应该从“基础设施补齐”逐步转向：
1. **Discovery**：告诉用户下一步值得玩什么
2. **Structure**：把 208 个成就组织成有路径感的体验
3. **Collectibility**：强化视觉资产、套装奖励、个人展示价值
4. **Trust**：让用户理解为什么解锁 / 为什么未解锁，并保证文档与系统状态一致

---

## 二、排序依据

本次 Top 10 主要按以下三个维度综合排序：

1. **用户体感价值**：用户是否能明显感知到更好玩、更有方向、更有成就感
2. **设计一致性**：是否符合调研结论（引导发现、优先体验感、克制胜过堆量）
3. **项目阶段匹配**：是否正好补上 AGPA 当前最明显的产品短板

---

## 三、Top 10 TODO

## 1. 推荐系统 2.0：从“近锁”升级为“下一步最值得玩的内容”

### 为什么排第一
AGPA 已经有大量成就和 progress nudge，但还缺少一个真正强的“下一步建议层”。

当用户面对 208 个成就时，最容易出现两个问题：
- 不知道先追哪个
- 看到一堆未解锁内容，但没有明确的下一步行动感

而调研中最值得贯彻的一条原则正是：

> 好的成就系统是引导玩家发现已有乐趣，而不是要求玩家刷成就。

### 建议目标
推荐层不应只做“离解锁最近的 5 个”，而应分成几类：
- **Near Win**：离解锁最近的 3 个
- **Discovery**：推荐一个尚未体验过的功能/玩法
- **Challenge**：推荐一个略高难但成就感强的目标
- **Surprise**：推荐一个隐藏/彩蛋型目标，只给 hint 不剧透

### 价值
这是最直接提升**日常回访率、探索欲、系统可玩性**的功能。

---

## 2. Questline / 成就旅程线：把 208 个成就从目录变成成长路径

### 为什么重要
现在 AGPA 已经有 set、title、badge，但从产品感知上仍更像“大型成就目录”，而不是“几条可以持续推进的成长路线”。

### 建议方向
将成就组织成数条清晰的旅程线，例如：
- **Bug Hunter**：调试、自修复、修 bug
- **Toolsmith**：MCP、hook、config、工具接入
- **Builder**：写文件、改代码、生成成果
- **Night Shift**：长会话、夜间、高强度耐力
- **Polyglot**：多语言、多栈探索

### 完成标准建议
- 每条线有明显的开始 / 中段 / 终章
- 以“体验核心乐趣”为目标，而不是要求全收集
- 终点给更强感知的奖励，如 title、profile flair、展示样式

### 价值
这会把 AGPA 从“很多 achievement”推进成“有 RPG 成长感的 achievement system”。

---

## 3. 成就作者工具链正式化：linter + simulator + preview + explain

### 为什么值
AGPA 已经有：
- YAML 作为权威源
- 规则审计
- LLM 审计脚本
- 大量测试

说明项目已经接近一个“成就开发平台”，差的不是底层能力，而是一个顺手的**内容生产工作流**。

### 建议能力
1. **单成就 dry-run**：输入 achievement id + 事件样本，直接判断会不会解锁
2. **描述-条件一致性 preview**：实时显示描述与条件是否匹配
3. **测试模板生成**：新成就自动生成最小触发测试 scaffold
4. **explain trace**：输出该成就命中了什么、还差什么

### 价值
下一阶段的瓶颈会越来越偏向**内容质量和内容迭代效率**。这类工具是长期杠杆。

---

## 4. 像素画 icon 先做高价值试点，而不是直接全量铺开

### 为什么值
视觉收藏欲是成就产品的核心体验之一，而不仅是装饰项。

当前像素画资产仍是明确的未完成项，说明视觉体系还有很大提升空间。

### 不建议的做法
不要一口气追求“208 张一次做完”。

### 更优方案
先做一批最值得落地的高价值 icon：
- 新手线 10~15 个
- 最常见 / 最有代表性的 10 个
- 最高稀有度 5~10 个
- 1~2 个完整 set 作为试点

### 价值
这是最能快速提升“产品完成度”“收藏感”“分享观感”的动作之一。

---

## 5. 套装奖励升级：从 title/badge 走向 Profile Cosmetics

### 为什么值
现在 set 完成后的奖励链路已经打通，但 title/badge 仍偏文字化、说明化。

下一步最自然的增强是把 set reward 升级成更强的视觉身份：
- profile 背景
- hero 边框
- 展示柜皮肤
- accent color / flair

### 建议优先级
先从两项体感最强、开发相对可控的做起：
1. Profile hero 背景样式
2. 展示柜边框 / 稀有度皮肤

### 价值
让“完成一套”不只是多一个标签，而是真正改变用户的个人资料外观。

---

## 6. 成就可解释层：Why unlocked / Why not unlocked

### 为什么值
当系统扩展到 208 个成就、12 条件类型时，用户一定会开始问：
- 我明明做了，为什么没解锁？
- 这个进度到底怎么算的？
- 这是没触发，还是没追踪到？

### 建议能力
为成就详情增加“Why / Why not”面板：
- 已命中的条件
- 当前计数/进度
- 还差多少
- 当前窗口范围
- 哪些事件被 filter 排除了
- hidden 成就只给 hint 级解释

### 价值
这不是单纯给开发者的调试工具，而是**用户信任层**。

---

## 7. 下一波内容扩展走“精品 20~30 个”，不要追机械扩容

### 为什么值
208 个成就已经很多。继续单纯扩容，边际价值会快速下降。

调研结论非常清晰：
- 好成就是引导体验，而不是强迫刷量
- 克制比堆量更重要
- 要避免纯 RNG、无意义刷量和高错过风险

### 建议优先补的内容类型
1. **Emergent / sandbox 行为型**：让用户觉得“原来这也算”
2. **Challenge ladder**：阶梯式挑战，而不是简单加大数字
3. **Playstyle branching**：不同风格走出不同路径
4. **Narrative hidden**：更有故事感的隐藏成就

### 价值
精品内容比总量更能定义 AGPA 的品牌风格。

---

## 8. 本地优先的 Daily / Weekly Challenges

### 为什么值
永久成就是收藏系统，周期挑战则是回访系统。两者功能不同。

AGPA 当前更偏累计型成就系统，而要进一步提升长期留存，适合增加：
- 每日轻挑战
- 每周中等挑战
- 每月主题挑战包

### 设计原则
- 本地优先
- 不强依赖云端
- 不做重度 FOMO
- 奖励可以是 XP、临时 flair、challenge badge、回顾卡素材

### 价值
增强“今天打开 AGPA 看看有什么可玩”的动机。

---

## 9. 分享从“静态卡片”升级到“回顾叙事”

### 为什么值
AGPA 已经具备分享卡片和导出能力，但如果要进一步提升传播性，下一步不应该只是换更多模板，而应该支持分享“故事”。

### 建议方向
- 本周回顾
- 本月成就报告
- 稀有成就故事卡
- 你是哪类 Agent 玩家画像
- 某条 Questline 完成海报

### 价值
把 AGPA 从“自己看着爽”推进到“别人也一眼觉得有趣”。

---

## 10. 真相源治理：文档数字同步 + 真实场景 smoke 验证自动化

### 为什么值
随着系统规模扩大，手工维护“成就数 / 测试数 / 条件类型 / CLI 命令数 / set 数”等数字已经开始出现漂移风险。

### 建议能力
- 关键数字改为脚本生成或自动校验
- 增加 docs verify / docs doctor 一类的检查命令
- 补充真实 event.log / hook / dashboard 的 smoke 流程，而不只依赖 unit test

### 价值
这件事不显眼，但它会直接影响：
- 项目可信度
- 文档可维护性
- 未来 roadmap 判断准确性

---

## 四、如果只能先做 3 个

### 产品价值优先
1. 推荐系统 2.0
2. Questline / 成就旅程线
3. 像素画试点 + 套装视觉奖励

### 长期工程杠杆优先
1. 成就作者工具链
2. 推荐系统 2.0
3. 可解释层 + 真相源治理

---

## 五、明确不建议当前优先投入的方向

## 1. 继续单纯追求更多成就数量
203 个已经足够大。若 discovery、questline、explain 还没跟上，继续扩容的边际价值会下降很快。

## 2. 现在就优先做全球社区 / 联网稀有度
在用户量还不足以支撑动态稀有度时，优先级不应高于本地体验和内容组织。

## 3. 继续把主要精力投入到“再补更多工具支持”
5 工具双通道覆盖已经很强。下一阶段更大的产品价值来自玩法设计和回访闭环，而不是横向继续铺接入面。

---

## 六、总结

AGPA 的下一个拐点，不是“更像一个追踪器”，而是：

> **更像一个有成长路径、有收藏欲、有回访理由的游戏化产品。**

因此，下一阶段最值得做的，不是继续补“能不能做”，而是强化：
- **Discovery**：下一步玩什么
- **Structure**：这 208 个成就如何形成旅程
- **Collectibility**：为什么值得收藏和展示
- **Trust**：为什么解锁、为什么没解锁、系统状态是否可信

如果后续要继续细化，这份 Top 10 可以继续拆成：
- 4 周执行路线图
- 按价值 / 成本 / 风险的优先级矩阵
- 按产品 / 引擎 / Dashboard / 内容制作四条线并行推进的版本计划
