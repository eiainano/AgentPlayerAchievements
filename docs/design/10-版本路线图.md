## 二、MVP (v0.1) — "先跑起来"

### 目标

一条完整的"用户用 CC 发消息 → 成就解锁 → 终端弹窗 → Dashboard 查看"链路。

### 范围

| 维度 | 范围 | 备注 |
|------|------|------|
| **工具** | Claude Code 独占 | Kilo/Hermes/OC/OpenClaw 后面再加 |
| **成就** | 30 个 | 选型见下表 |
| **Evaluator** | CounterEvaluator + SequenceEvaluator + ThresholdEvaluator | Streak + Distinct 暂缓 |
| **展示** | 终端 ANSI 弹窗 + 对话内成就展示 | CC 内部展示 |
| **Dashboard** | 基础版：成就网格 + 已解锁/未解锁 | 无 profile hero / 展示柜 / 时间线 |
| **Event Capture** | CC Hook + MCP track + MCP poll | 3 种标准化 Pipeline |
| **全球统计** | ❌ 不做 | v1.0 再做 |
| **其他工具** | ❌ 不做 | v1.0 做 |

## 四、Future — "想象空间"

### v1.x 可能的方向

```
成就系统增强:
  - 赛季成就（Season Pass）
  - 成就挑战（限时成就，如"今天完成 50 个 task"）
  - 成就套装奖励视觉化（Profile 背景、边框、称号）

事件采集增强:
  - 文件系统 Watch（inotify/FSEvents 捕获 file.* 事件）
  - Git Hooks 集成（捕获 git.commit）
  - Agent SDK 插件（工具原生集成，无需 Hook/指令）

Dashboard 增强:
  - 成就分享卡片（PNG 导出）
  - 成就搜索 / 过滤 / 排序
  - 成就推荐（"还剩 3 个 task 到下一个成就！"）
  - 成就进度推送（poll() 时不只是展示，有动态提示）

社区:
  - 全球排行榜（opt-in）
  - 成就评论区 / 攻略
  - 用户自定义成就（YAML 编辑器 + 热加载）
```

### 不做的事（明确拒绝）

```
✗ 联网账户系统（agpa 永远是本地优先、可选 opt-in）
✗ 广告 / 第三方追踪
✗ prompt 内容 / 代码内容上传
✗ DAU/MAU 指标
```

## 五、版本号与发布策略

```
v0.1.0  MVP（第一版可跑通的实现）
v0.2.0  补齐成就到 60 个
v0.3.0  补齐成就到 90 个
v1.0.0  109 成就 + 5 工具 + Dashboard 完整

版本号规则：
  major: 破坏性变更（如 state.json 格式不兼容）
  minor: 功能新增（新成就、新工具、Dashboard 新页面）
  patch: 修复（条件评估 bug、Dashboard 渲染问题）

预发布标签：
  v0.1.0-alpha.1   内部测试
  v0.1.0-beta.1    邀请测试
  v0.1.0-rc.1      候选发布
  v0.1.0           正式发布

发布频率：
  MVP:  每 2-3 天一个 alpha，直到功能完整
  v1.0: 每周一个 beta
  稳定: 每月一个 minor
  修复: 按需 patch
```
